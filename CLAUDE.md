## Gambaran Proyek

Aplikasi e-money untuk lingkungan kampus. Flutter (frontend) berkomunikasi dengan backend Go (Gin) melalui REST API, dan langsung memanggil Firebase SDK untuk autentikasi.

**Fitur utama:**
- Autentikasi Firebase (Google Sign-In + Email/Password)
- 2FA: Email OTP (SMTP), Google Authenticator (TOTP), Push Notification (FCM)
- Dashboard saldo & transaksi
- Transfer, Top Up, Pembayaran QRIS, Deeplink merchant
- Riwayat transaksi

## Semua Endpoint API

File: `lib/core/constants/api_endpoints.dart`

| No | Method | Path | Auth | Fungsi |
|----|--------|------|------|--------|
| A.1 | POST | /v1/auth/verify-token | Publik | Login — Firebase ID Token → Backend JWT |
| A.2 | POST | /v1/auth/register | Publik | Registrasi user baru + kirim OTP email |
| A.3 | POST | /v1/auth/verify-email-otp | JWT | Verifikasi kode OTP email saat registrasi |
| A.4 | GET | /v1/auth/me | JWT | Ambil profil user yang sedang login |
| A.5 | PUT | /v1/auth/fcm-token | JWT | Simpan FCM device token |
| A.6 | POST | /v1/otp/send-firebase | JWT | Kirim OTP via notifikasi Firebase (FCM) |
| A.7 | POST | /v1/otp/send-email | JWT | Kirim OTP via email (SMTP) |
| A.8 | POST | /v1/otp/confirm | JWT | Verifikasi kode OTP (firebase/email) |
| A.9 | POST | /v1/otp/totp/register | JWT | Daftarkan TOTP (QR code + secret) |
| A.10 | POST | /v1/otp/totp/verify | JWT | Verifikasi kode TOTP & aktifkan |
| A.11 | GET | /v1/account | JWT | Ambil saldo & info akun e-money |
| A.12 | GET | /v1/account/transactions | JWT | Ambil 20 transaksi terakhir |
| A.13 | POST | /v1/payment/topup | JWT | Top up saldo (tanpa OTP) |
| A.14 | POST | /v1/payment/transfer | JWT | Transfer/pembayaran dengan verifikasi OTP |

**Format Response Standar:**
```json
{ "success": true, "message": "...", "data": { ... } }
{ "success": false, "message": "Pesan error", "error_code": "INVALID_OTP", "data": { ... } }
```

## Error Codes Penting

| Exception Flutter | Pemicu | Efek di UI |
|-------------------|--------|-----------|
| NetworkException | connectionError / connectionTimeout | "tidak ada koneksi" |
| UnauthorizedException | HTTP 401 (bukan INVALID_OTP) | redirect login |
| InvalidOtpException | HTTP 401, error_code=INVALID_OTP/INVALID_TOTP | shake + kosongkan form OTP |
| InsufficientBalanceException | error_code=INSUFFICIENT_BALANCE | info saldo tidak cukup |
| ServerFailure | success:false lainnya | tampilkan message dari backend |

## Alur Utama

### Registrasi Baru
```
RegisterPage → Firebase.createUser → getIdToken → POST /v1/auth/register
→ VerifyEmailPage → POST /v1/auth/verify-email-otp
→ Setup2FAPage → pilih metode → /2fa/<metode> (mode: setup)
→ HomePage
```

### Login Lama
```
LoginPage → Firebase.signIn → getIdToken → POST /v1/auth/verify-token
→ AuthNeedsVerification → /2fa/<metode-aktif> (mode: login)
→ HomePage
```

### Transaksi Transfer
```
HomePage → /transfer → /transfer/amount → /transfer/confirm
→ /pin → POST /v1/payment/transfer (dengan otp_code + otp_type)
→ /success → /home
```

### Top Up
```
HomePage → /topup → POST /v1/payment/topup (TANPA OTP)
→ /success → /home
```

## Setup Backend (Prasyarat Sebelum Flutter)

1. **MySQL** — buat database `emoney`, user, password
2. **Redis** — via Docker: `docker run --name redis-server -p 6379:6379 -d redis`
3. **Firebase** — buat project, enable Auth (Email+Google), download `service_account.json`
4. **Gmail SMTP** — enable 2FA, buat App Password
5. **Backend Go** — tambah `.env`, jalankan `go run main.go`
6. **Flutter Firebase** — `flutterfire configure` → generate `lib/firebase_options.dart`

### File .env Backend
```env
PORT=8080
DB_HOST=localhost  DB_PORT=3306  DB_USER=useremoney  DB_PASSWORD=Password#123  DB_NAME=emoney
REDIS_HOST=localhost  REDIS_PORT=6379  REDIS_PASSWORD=
JWT_SECRET=your-super-secret-jwt-key-change-this
FIREBASE_CREDENTIALS_PATH=firebase_service_account.json
SMTP_HOST=smtp.gmail.com  SMTP_PORT=587  SMTP_USER=email@gmail.com  SMTP_PASSWORD=app-password
OTP_EXPIRY_MINUTES=5
```

## Catatan Penting

- `baseUrl` pakai IP lokal. Emulator Android: `10.0.2.2`, iOS: `localhost`
- Field JSON dari GORM Go: `"ID"` (kapital) dan `"CreatedAt"` (kapital) — baca fleksibel di `fromJson`
- Transfer butuh OTP. Top Up TIDAK butuh OTP.
- Setelah login, `AuthBloc` emit `AuthNeedsVerification` → paksa 2FA sebelum masuk home
- `kAuthVerified=false` = sesi tidak valid meski JWT masih ada
- QRIS saat ini simulasi (merchant data hardcoded, bukan parsing payload QRIS asli)
